# Баг-репорт: Сессия не завершается полностью при нажатии Log out

**ID:** BR-004
**Статус:** New
**Приоритет:** High
**Серьезность:** Major
**Модуль:** Управление сессией / Безопасность

**Заголовок:** После Logout можно вернуться к защищенным страницам через кнопку Back браузера

**Окружение:**
- ОС: Windows 11 Pro
- Браузер: Яндекс Браузер 25.10.2.1176
- Устройство: Ноутбук Thunderrobot
- URL: https://demo.prv029.com/documented/24/index.php
- Дата/время обнаружения: 15.12.2025 19:36 (MSK)

**Предусловие:** 
1. Пользователь авторизован в системе (email: qa_test_nikita@mail.ru)
2. Открыта домашняя страница "Home"

**Шаги для воспроизведения:**
1. Убедиться, что отображается страница "Home" и надпись "Signed in as qa_test_nikita@mail.ru"
2. Нажать кнопку "Log out" в навигационном меню
3. Убедиться, что отобразилась страница авторизации (Login)
4. Нажать кнопку "Назад" (Back) в браузере
5. Наблюдать результат

**Фактический результат:**
1. После нажатия "Log out" отображается страница авторизации (✅)
2. При нажатии кнопки "Назад" (Back) в браузере:
   - Происходит возврат на домашнюю страницу "Home"
   - Отображается надпись "Signed in as qa_test_nikita@mail.ru"
   - Пользователь снова видит защищенную страницу без повторной авторизации
   - Навигационное меню доступно (Home, Profile, Edit Profile и т.д.)
3. Сессия НЕ завершена полностью
4. Кэш браузера сохраняет авторизованное состояние
5. Пользователь может продолжать доступ к защищенным страницам через историю браузера

**Ожидаемый результат:**
1. После нажатия "Log out":
   - Сессия должна быть полностью завершена на сервере
   - Все сессионные данные должны быть очищены
   - Cookie аутентификации должны быть удалены или помечены как недействительные
2. При попытке вернуться через кнопку "Назад":
   - Должно происходить перенаправление на страницу авторизации
   - Или показываться сообщение "Сессия завершена"
   - Доступ к защищенным страницам должен быть запрещен
3. После Logout пользователь должен проходить полную повторную аутентификацию для доступа
